## Responsibilities & Scope

### 1. Corporate Registration & Governance
- **Bangladesh RJSC Registration:** Ensure company maintains valid registration with Registrar of Joint Stock Companies and Firms (RJSC), Bangladesh
  - File annual returns and financial statements with RJSC
  - Track renewal dates and statutory filing deadlines
  - Maintain certified copies of registration certificates
  - Ensure board meeting minutes and corporate resolutions are documented

- **Trade License Compliance:**
  - Secure and renew trade license from City Corporation/Municipality
  - Update business activities to include "fintech" and "pledge collection platform" once applicable
  - Maintain compliance with local business regulations

- **Tax & Financial Reporting:**
  - Coordinate with accountant for VAT, corporate income tax filings
  - Maintain documentation for audit trails
  - Ensure financial transparency for investor and regulatory requirements

### 2. Intellectual Property Protection
- **Trademark Registration:**
  - File "SalamiPay" trademark with Bangladesh Intellectual Property Office (BIPO)
  - Secure trademark in relevant classes (financial services, software, payments)
  - Monitor international trademark registration (WIPO Madrid Protocol for expansion markets)

- **Domain Protection:**
  - Maintain ownership and WHOIS privacy for `salamipay.com`, `salamipay.app`, `app.salamipay.com`
  - Monitor domain expiration dates (renewals 60+ days prior)
  - Implement DNSSEC and registrar locks
  - Protect against domain hijacking and cybersquatting

- **Software IP:**
  - Maintain documentation of proprietary code architecture
  - File copyright notices for custom code (not dependencies)
  - Conduct periodic IP audits

### 3. Terms of Service & Privacy Policy Maintenance
- **Terms of Service (ToS):**
  - Define user rights, limitations of liability, dispute resolution
  - Include caps on platform liability
  - Clarify status as pledge-tracking platform (not payment processor)
  - Document payment processor disclaimers (when integrated with bKash/Nagad/Stripe)
  - Include acceptable use policy, content moderation clauses
  - Review quarterly or upon regulatory changes

- **Privacy Policy:**
  - Detail data collection, processing, storage practices
  - Define data retention periods and deletion procedures
  - Explain third-party integrations (Supabase, payment providers)
  - Include consent mechanisms and user rights
  - Maintain versioning with change logs
  - Review semi-annually or upon integration changes

- **Cookie & Tracking Policy:**
  - Document all cookies and tracking technologies
  - Provide granular consent mechanisms for GDPR/international users
  - Maintain cookie audit trail

### 4. Data Protection & Compliance
- **Bangladesh ICT Act (2013) Compliance:**
  - Implement data protection measures per Section 79
  - Develop data security standards and breach response procedures
  - Maintain data handling documentation
  - Report critical data breaches to Bangladesh Telecommunication Regulatory Commission (BTRC) if applicable

- **GDPR Compliance (International Users):**
  - Implement Data Processing Agreement (DPA) with Supabase
  - Establish legal basis for data processing (consent, legitimate interest)
  - Develop Data Subject Access Request (DSAR) procedures
  - Maintain data processing impact assessments
  - Implement data minimization and purpose limitation
  - Document right to erasure ("right to be forgotten") mechanisms

- **Data Localization:**
  - Ensure compliance with any Bangladesh data residency requirements
  - Document Supabase infrastructure locations (PostgreSQL hosting)
  - Assess cross-border data transfer mechanisms

### 5. Fintech Regulatory Landscape
- **Bangladesh Bank Guidelines:**
  - Monitor Bangladesh Bank's Payment System Policy Framework
  - Track Digital Financial Services (DFS) guidelines for mobile money integration
  - Ensure compliance with Know Your Customer (KYC) requirements if accepting payments
  - Maintain Anti-Money Laundering (AML) and Counter-Terrorist Financing (CTF) documentation

- **Payment Service Provider (PSP) Licensing:**
  - Clarify whether pledge-tracking requires PSP license (legal research needed)
  - If accepting payments via bKash/Nagad/Stripe, ensure partnered providers hold valid PSP licenses
  - Monitor license requirements for future payment integration
  - Document liability limitations as non-PSP platform

- **e-Commerce & Digital Transaction Guidelines:**
  - Comply with Bangladesh Bank's e-Commerce guidelines for digital payments
  - Maintain secure transaction logs and audit trails
  - Implement encryption standards for sensitive data

- **Regulatory Monitoring:**
  - Subscribe to Bangladesh Bank circulars and notifications
  - Attend regulatory workshops and compliance briefings
  - Maintain communication channel with regulatory affairs team

### 6. User Agreement & Consent Management
- **Explicit Consent Mechanisms:**
  - Implement consent checkboxes during user registration
  - Require separate consent for terms, privacy, and cookie policies
  - Maintain audit logs of user consent with timestamps
  - Allow granular opt-in/opt-out for optional features

- **Minor Protection:**
  - Age verification (confirm users 18+) at registration
  - Parental consent mechanism for minors (if applicable)
  - Restrict minors' participation in certain financial activities

- **Withdrawal of Consent:**
  - Provide easy mechanisms for users to withdraw consent
  - Document consent withdrawal procedures and timelines
  - Handle data retention after consent withdrawal

### 7. Liability & Disclaimer Frameworks
- **Platform Liability Limitations:**
  - Cap liability to lesser of amount contributed or fixed amount (e.g., 1000 BDT)
  - Exclude liability for indirect, incidental, or consequential damages
  - Clarify no liability for third-party payment processor failures
  - Exclude liability for user-contributed content

- **Merchant Liability Disclaimer:**
  - Clarify SalamiPay does not guarantee event organizer fulfillment of pledges
  - State that contributors assume risk of event cancellation or non-delivery
  - Provide dispute resolution process (not guarantee of refund)

- **Security Disclaimer:**
  - Limit liability for unauthorized access despite reasonable security measures
  - Recommend user account security best practices
  - Clarify liability limits in case of data breach

### 8. Employment Law & Contracts
- **Employment Agreements:**
  - Develop standard employment contracts compliant with Bangladesh Labour Rules 2015
  - Include probation clauses, non-compete, confidentiality, IP assignment
  - Document salary, benefits, leave entitlements
  - Ensure compliance with minimum wage regulations

- **Workplace Policies:**
  - Develop code of conduct, anti-harassment, anti-discrimination policies
  - Create performance management and termination procedures
  - Maintain compliance with occupational safety regulations

- **Statutory Compliance:**
  - Register with Department of Labour for establishments with 10+ employees
  - Maintain employee provident fund contributions (if applicable)
  - Ensure statutory health insurance coverage
  - Track statutory leave entitlements (annual, sick, special leave)

### 9. Contractor & Vendor Agreements
- **Contractor Agreements:**
  - Develop standard independent contractor agreements
  - Include IP assignment, confidentiality, non-compete clauses
  - Define scope of work, deliverables, payment terms
  - Clarify liability limitations and dispute resolution

- **Service Provider Agreements:**
  - Maintain Data Processing Agreements (DPA) with Supabase
  - Establish agreements with payment providers (bKash, Nagad, Stripe)
  - Document terms with hosting, email, analytics, and monitoring providers
  - Include liability, indemnification, and breach notification clauses

- **Vendor Audit:**
  - Review vendor compliance with data protection and security standards
  - Monitor vendor service level agreements (SLAs)
  - Document vendor risk assessments

### 10. NDA & Confidentiality Management
- **Internal NDAs:**
  - Require all employees to sign confidentiality agreements at hiring
  - Include detailed definition of confidential information
  - Set confidentiality duration (during employment + 2–3 years post-termination)
  - Include remedies and specific performance clauses

- **External NDAs:**
  - Develop standard NDA templates for investor discussions, partnership negotiations
  - Require signed NDAs before sharing sensitive business information
  - Maintain a register of executed NDAs with expiration dates

- **Trade Secrets Protection:**
  - Implement access controls and data encryption
  - Limit confidential information to need-to-know basis
  - Document proprietary algorithms, business models, user database
  - Maintain audit logs of access to sensitive information

### 11. Dispute Resolution Procedures
- **Internal Escalation Process:**
  - Level 1: Customer support team (support@salamipay.com) — 7-day response
  - Level 2: Legal Agent — investigation and mediation — 14-day response
  - Level 3: Master Agent & Board — final decision — 30-day response

- **Mediation & Arbitration:**
  - Establish alternative dispute resolution (ADR) procedures
  - Offer mediation before litigation (cost-effective, faster resolution)
  - Include mandatory arbitration clause in ToS for disputes under specified amount (e.g., 100,000 BDT)
  - Specify arbitration venue in Bangladesh (Dhaka preferred)

- **Refund & Reversal Policy:**
  - Define clear criteria for contribution refunds (event cancellation, user error, etc.)
  - Document refund processing timelines (7–14 days for pledges)
  - Establish fraud investigation and chargeback handling procedures

- **Complaints Registry:**
  - Maintain formal log of user complaints with timestamps
  - Track resolution status and closure dates
  - Report complaints to regulatory bodies if required

### 12. Open Source License Compliance
- **Dependency Audit:**
  - Maintain inventory of all open-source dependencies
  - Current stack: Next.js 16 (MIT), React 19 (MIT), TypeScript (Apache 2.0), Tailwind CSS 4 (MIT), Supabase SDK (Apache 2.0)
  - Review licenses for compatibility with SalamiPay proprietary code

- **License Compliance:**
  - MIT & Apache 2.0 allow commercial use with proper attribution
  - Maintain ATTRIBUTION.md or LICENSE file with all open-source credits
  - Include license notices in source code headers where required
  - Conduct quarterly audits for new dependencies

- **GPL/Copyleft Avoidance:**
  - Ensure no GPL-licensed code is directly included in proprietary codebase
  - Use GPL libraries only in clearly separated modules (if necessary)
  - Document any GPL-licensed optional dependencies

- **Compliance Checklist:**
  - [ ] ATTRIBUTION.md updated with all licenses
  - [ ] package.json reviewed for license compliance
  - [ ] No GPL code in core proprietary modules
  - [ ] License notices in source files where required

### 13. Cookie & Tracking Consent (GDPR)
- **Cookie Banner Implementation:**
  - Display cookie consent banner on first visit (prior to non-essential cookies)
  - Allow granular selection (essential, analytics, marketing, performance)
  - Store consent preferences in local storage or database
  - Honor "Do Not Track" browser signals

- **Consent Preference Management:**
  - Provide easy access to privacy center/consent management
  - Allow users to modify consent preferences at any time
  - Document consent with timestamps and IP addresses
  - Honor withdrawal of consent immediately

- **Cookie Categories:**
  - Essential: Session, CSRF tokens, authentication (required)
  - Analytics: Google Analytics, Supabase analytics (optional, GDPR-compliant)
  - Performance: CDN cookies, caching (optional)
  - Marketing: Third-party tracking (if any) — explicitly opt-in

- **Third-Party Integrations:**
  - Document all third-party scripts and their cookie usage
  - Include data sharing terms with third parties in privacy policy
  - Maintain Data Processing Agreements with analytics/tracking providers
