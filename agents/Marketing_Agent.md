### 4. Growth Hacking & Viral Loops

**Objective:** Activate exponential user growth through product-driven and community-driven virality.

**Viral Loop Mechanisms:**

**Loop 1: Event Sharing**
- User creates event → Generates shareable link → Shares with network → Friends join and share → Network effect
- Incentive: Prominent share buttons, pre-filled social media messages, email templates
- Tracking: UTM parameters on all shared links
- Target: Each event shared average 5–10 times minimum

**Loop 2: Referral Program**
- New user signup → Refers 3+ friends → Gets feature unlock/badge/small reward
- Incentive: Premium feature access, custom branding, early feature access
- Tracking: Unique referral codes per user, attribution table
- Target: 30% of new signups come from referrals

**Loop 3: Social Proof & FOMO**
- Display trending events on homepage ("Trending Now", "Recently Funded")
- Show real-time contribution counts
- Email notifications: "Your friend just contributed to Event X"
- Push notifications (in-app): "500 people already joined this event!"
- Target: Increase event awareness and FOMO conversions by 25%

**Loop 4: Community Ambassador Program**
- Identify power users (event organizers with 50+ participants)
- Offer ambassador status: early access to features, co-marketing, small stipends
- Ambassadors: Drive growth in their communities, share success stories
- Target: 50 ambassadors recruiting 20+ new users each in Year 1

**Growth Channels Priority:**
1. Organic social (Facebook + WhatsApp) — highest ROI for Bangladesh market
2. Referral (in-app + word-of-mouth)
3. Content marketing (SEO + blog)
4. Paid social (Facebook/Instagram ads) — if budget available
5. Partnerships (complementary platforms, NGOs, community groups)
6. PR (startup media, Bangladeshi tech press)

**Metrics:**
- User acquisition cost (UAC)
- Viral coefficient (K-factor)
- Month-over-month user growth
- Event creation rate
- Contribution per event (average)
- Return on ad spend (ROAS) if paid campaigns active

### 8. Brand Voice & Messaging Guidelines

**Brand Personality:** Trustworthy, Community-Focused, Modern, Cultural, Accessible

**Voice Characteristics:**
- **Warm & Personal:** "We're part of your community, not just a tech company"
- **Clear & Simple:** Explain complex financial concepts in everyday language
- **Empowering:** Focus on what users CAN do (organize, connect, collect)
- **Bengali-Aware:** Respect cultural nuances; avoid tone-deaf phrasing
- **Inclusive:** Address diverse user segments (families, students, businesses)

**Key Messaging:**

**Primary Message (Elevator Pitch):**
"SalamiPay makes organizing group contributions simple and secure. Perfect for salami, donations, celebrations, and community fundraising—right from WhatsApp and your phone."

**Secondary Messages:**
1. **For Event Organizers:** "No more spreadsheets, no more confusion. SalamiPay tracks every contribution in real-time."
2. **For Contributors:** "Contribute in seconds. Share instantly. Know exactly who's paying and how much."
3. **For Families:** "Organize family events, celebrations, and donations with trust and transparency."
4. **For Communities:** "Build stronger communities through transparent, organized collective contributions."

**Tone Examples:**

| Context | Do | Don't |
|---------|----|----|
| Onboarding new user | "Welcome! Let's set up your first event in 2 minutes." | "Input required data into form fields." |
| Error message | "Oops, something went wrong. Here's how to fix it." | "Error 404: Contribution not found." |
| Success message | "Amazing! 50 people contributed to your event!" | "Data successfully persisted to database." |
| Explaining fees | "A tiny percentage keeps SalamiPay running." | "Platform takes 2% cut." |
| Cultural context | "Perfect for Pohela Boishakh collections, Eid donations, wedding expenses..." | "Event-based payments" |

**Banned Phrases:**
- "Disrupt" (overused startup cliché)
- "Blockchain" or "crypto" (unless specifically relevant; Bangladeshi users skeptical)
- "Just" (minimizes value: "Just pay", "Just organize")
- Jargon without explanation (fintech, SaaS, API, etc.)
- Overly formal corporate language

**Hashtags:**
- Primary: #SalamiPay #GroupPayments #BDTech
- Community: #CommunityLove #TogetherWeGrow #BangladeshiStartup
- Campaign-specific: (created per campaign)

### 11. Referral Program

**Objective:** Incentivize users to recruit friends, creating exponential growth.

**Program Structure:**

**Participant Tiers:**
- **All Users:** Basic referral link (unique per user)
- **Power Users (50+ contributions):** Enhanced rewards
- **Ambassadors (200+ referrals):** Special status, exclusive benefits

**Referral Mechanics:**

1. **In-App Referral Button:**
   - Generate unique referral code per user (alphanumeric, 6–8 chars)
   - QR code with link (salamipay.com/ref/[CODE])
   - Pre-written WhatsApp/Facebook message templates
   - Copy-to-clipboard link
   - Tracking: Every click logged, every signup attributed

2. **Attribution & Tracking:**
   - Cookie-based (30-day default)
   - UTM parameter tracking (utm_source=referral, utm_medium=[CODE])
   - Database: referrals table with referrer_id, referee_id, signup_date, reward_claimed

3. **Reward Tiers:**
   - Referrer: Gets reward after referee completes first contribution
   - Rewards can be: Feature unlock (custom branding), app badge, early access, or in future: small discount/credit
   - **Milestone Bonuses:** 10 referrals = special badge, 25 = featured on leaderboard, 50 = ambassador status

4. **Viral Incentives:**
   - Referee also gets small welcome reward (transparency, fairness)
   - Leaderboards: Monthly top 10 referrers highlighted on social media
   - Special campaigns: "Refer a Friend in Ramadan" → doubled rewards

**Messaging Examples:**
- Referral email: "Get $X credit when your friends sign up. Share your link: [URL]"
- WhatsApp: "I'm using SalamiPay to organize payments easily. Check it out: [LINK]"
- Social media: "3 friends just joined my event on SalamiPay! Use my link to get started: [LINK]"

**Metrics:**
- Referral signup count
- Referral conversion rate (clicks → signups)
- Cost per referral (if incentivized)
- Viral coefficient (referred users who also refer)
- Referral cohort LTV vs. organic
